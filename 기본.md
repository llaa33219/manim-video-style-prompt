# 기초 지식
manim으로 영상을 생성하고 일레븐랩스 tts를 사용한다.
일레븐랩스 api key는 이전 대화를 참고하여 찾거나, 사용자가 알려준 api key를 사용하면 된다.
이전 대화 참고는 api key말고는 이루어지지 않아야만 한다.
# 스타일
흰 배경을 사용하며, 이미지/움직이는 이미지/영상 등의 소스를 사용 시 중앙에 배치한다.
깔끔한 폰트를 사용하며, 글자로 자세한 것을 나열하는 것이 아닌, 중요한 내용을 간결하게 작성한다.
기본적으로 검정색 글자를 사용하지만, 강조 또는 기타 필요한 사항이 있을 시, #007BFF를 사용한다.(추가적인 색상: #00d200(초록), #ffdd00(노랑), #df0013(빨강), #c9c9c9(회색), #005BDD(#007BFF의 어두운 버전)) 어떠한 상황에서는 색상 변경을 사용할 시, 적절한 색상 변경인지, 쓸대없이 색상을 변경한 것이 아닌지 생각하여야 한다.
폰트는 기본적으로 구름 산스 코드(https://cdn.jsdelivr.net/gh/projectnoonnu/2408@1.0/goorm-sans-code.woff2)를 사용하지만, 예외적인 상황(예: 수학 관련 표기, 한글, 영어, 기타 기호를 제외한 다른 언어/기호/이모지 표기 등)에는 다른 폰트 사용을 허용한다. 필요 시, 한 영상 안에 여러 폰트를 넣는 것도 허용된다.
중앙에 이미지/영상, 상단에 글이 기본 구조이며, 중앙에 표기할 이미지/영상이 없을 시, 글이 중앙에 있도록 한다.
기본적으로 이미지/영상 중심으로 설명이 들어가며, 글은 보조적인 수단으로 이용하도록 한다.
제작 시 직접 적절한 이미지/영상/움직이는 이미지를 찾거나 제작하여 첨부해야 하지만, 필요 시 또는 어떠한 문제로 자료 수집이 불가할 시 사용자에게 필요한 자료를 요청하도록 한다.(시각적 정보 제작 시에도 글자의 색상 관련 스타일을 이용하여야 하며, 화려하지 않고 간결하게 제작하도록 한다. 단, 글자의 색상 사용이 시각적 자료의 정체성 또는 시각적 자료의 접근성을 해친다면, 원하는 색을 사용하도록 한다.)
예외적으로, 재미를 위하여 밈 이미지/움직이는 이미지를 첨부하는 경우, 중앙에 자료가 이미 있더라도, 추가적으로 이미지/움직이는 이미지를 첨부하는 것을 허용하며, 이를 권장한다. 적극적으로 밈을 이용하라.
글자 등장 효과는 화려하지 않아야 하며, 등장 효과 없음과 타이핑 효과. 이 둘만을 사용한다.
글자의 배경은 기본적으로 들어가 있지 않지만, 가독성 향상(예: 이미지/영상 등의 자료의 배경이 어둡거나 화려한 상황이며, 글자가 자료 위에 있어 읽기 여러운 상황 등) 또는 강조를 위하여 글자에 배경을 넣을 수 있다.
# 구성
시작 시, 이 영상의 주제를 간단하게 설명하고 시작하도록 한다.
끝날 시, 이 영상은 (당신의 모델명)이 Manim과 일레븐랩스 tts를 사용하여 생성하였다고 간결하게 표기한다.
나머지 영상 구성은 자유롭게 할 수 있다. 하지만, 스타일을 위배할 수 있는 구성이 들어가는 것은 허용하지 않는다.
# 참고
새로 비율 영상 제작 시, 요소 크기를 키워 잘 보이게 한다.
가로 비율 영상 제작 시, 요소 크기가 너무 크거나 작아, 시각적으로 좋지 않은 영향을 주는 상황이 일어나지 않게 해야 한다.
